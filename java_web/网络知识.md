# Tomcat
Web 应用服务器，专门用来运行 Web 程序的。

启动一个服务，端口是8080，浏览器可以通过url访问服务（相当于对外开放一个窗口，里面存放外部可以访问的资源）

- bin：启动 / 关闭 Tomcat 服务的命令
- conf：配置文件
- lib：jar 包
- logs：日志
- temp：临时文件
- webapps：存放程序 （webapps/ROOT/index.jsp）
- work：JSP 文件转换之后的 Servlet 文件

# Servlet
java web提供的后端组件

Servlet 是一个接口，让 Java 具备接收网络请求，并且做出响应的功能

```java
@WebServlet("/test")
public class MyServlet implements Servlet {
    // 另外实现4个方法
    @Override
    public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {
        String idStr = servletRequest.getParameter("id");
        System.out.println("接收到了客户端参数：" + idStr);
        servletResponse.getWriter().write("Hello World");
    }
}

public class HelloServlet extends HttpServlet {
    // 相当于service拆成两个分支
    @Override
    public void doGet();
    
    @Override
    public void doPost();
}
```

# 文件结构

前端文件放置位置 src/main/webapp/

```
src/main/webapp/
├── WEB-INF/          # 受保护的目录，不能直接通过 URL 访问
│   ├── web.xml
├── index.jsp         # 首页
```
- WEB-INF 目录下的文件无法直接通过浏览器访问，适合放配置文件、JSP 模板等
- 所有放在 webapp/ 目录下的文件都会被打包到 WAR 包中

## 中文乱码问题

IDEA Tomcat 运行配置

1. 选中你当前的 Tomcat 服务配置，切换到「Server」标签页
2. 找到「VM options」（虚拟机选项），添加以下参数：
```
-Dfile.encoding=UTF-8
-Dsun.stdout.encoding=UTF-8
-Dsun.stderr.encoding=UTF-8
```

## HttpServletRequest

- getParameter	从 HTTP 请求中获取客户端提交的参数（如表单数据、URL 参数）
- setCharacterEncoding	设置请求 / 响应的字符编码，解决中文乱码问题
- URL	完整的资源定位地址，包含协议、主机、端口、资源路径
- URI	资源的唯一标识，这里指具体的资源文件名
- Session	Java Web 会话对象，用于跨请求保存用户状态（如登录信息）
- Request	HTTP 请求对象，封装了客户端的所有请求信息
- JSP	动态网页技术，是 HTML 和 Java 代码的混合，最终会编译为 Servlet 运行
